Download GStreamer

In general, you should find packages that were specifically made for your distribution or OS. Do not compile from source yourself unless you are certain you have to or want to. Do not compile from Git unless you really need some specific feature that is not released yet, or want to help out developing GStreamer.

Ready-made packages for your platform

Linux distributions provide their own packages of GStreamer. You should find these in your distro's package repository.

Note that some distributions split the GStreamer plugins up further than the upstream sources. Additionally, some distributions do not include the "bad", "ugly" and gst-libav packages in their main repository, for legal reasons.

Packages for various architectures are available, see the OpenBSD ports page.

MSI installers for Windows can be found here. They are provided and maintained by the GStreamer project.

A .framework for Mac OS X can be found here. It is provided and maintained by the GStreamer project.

Binaries for GStreamer on Android that integrate into the Android NDK can be found here. They are provided and maintained by the GStreamer project.

Compiling from the source code

For the stable source code releases, check our modules page, or go straight to our source download directory.
