Welcome, testers!

GStreamer has an extensive test suite that you can use and improve. Sometimes though, nothing beats testing by a human. Providing detailed bug reports for defects you encounter is always appreciated.

When reporting a bug, make sure that:

  • The bug affects the latest stable supported release
  • Can be reproduced with the development version
  • Has an adequate stack trace (in the case of crashers), including debug symbols. See GNOME's guide on getting traces.

Bug Lists

You can file a new bug report here. See the GStreamer bugzilla overview for a nice dynamic bird-eye's view of the existing bug reports on GStreamer. You can use that page to drill down into what interests you most, or use the Search form to find based on keywords.

In addition to that, here are some examples of useful queries:

  • All unresolved bugs
  • All unresolved bugs not assigned to a component
  • Fixed bugs which need to be reassigned to the right milestone
  • All bugs changed within the last 24 hours, 48 hours, 72 hours, 1 week, 2 weeks
